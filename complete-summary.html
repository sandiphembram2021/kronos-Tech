Kronos OS - Complete Technical Summary

Kronos OS is a modern, 64-bit operating system built from scratch using C and x86_64 Assembly. This document provides a comprehensive overview of the system architecture, features, and development status.

System Architecture

Kernel Architecture

Type: Monolithic Kernel
Mode: 64-bit Long Mode
Memory Model: Flat memory model with paging

Boot Process

Bootloader: GRUB2 Multiboot2
Entry Point: 64-bit kernel entry
Initialization: GDT, IDT, Memory setup

Memory Management

Physical Memory: Bitmap allocator
Virtual Memory: 4-level paging
Heap: 8MB kernel heap

Interrupt System

IDT: 256 interrupt descriptors
PIC: Remapped to IRQ 32-47
Handlers: Timer, Keyboard, Exceptions

User Interface

Command Line Interface

  • Shell: Kronos Shell (ksh) with command parsing
  • VGA Driver: 80x25 text mode with color support
  • Keyboard: PS/2 keyboard driver with scancode translation
  • Commands: Built-in system commands and utilities

Graphical User Interface

  • Desktop Environment: Ubuntu-inspired interface
  • Window Manager: Overlapping windows with decorations
  • Taskbar: Application launcher and system tray
  • Mouse Support: PS/2 mouse with cursor rendering
  • Graphics: Framebuffer-based 2D graphics

Built-in Applications

System Applications

Technical Specifications

Memory Layout

Virtual Memory Layout: 0x0000000000000000 - 0x00007FFFFFFFFFFF : User Space (128 TB) 0xFFFF800000000000 - 0xFFFFFFFFFFFFFFFF : Kernel Space (128 TB) Physical Memory: 0x0000000000000000 - 0x00000000000FFFFF : Reserved (1 MB) 0x0000000000100000 - 0x00000000001FFFFF : Kernel (1 MB) 0x0000000000200000 - 0x0000000000A00000 : Heap (8 MB) 0x0000000000A00000 - ... : Available Memory

System Limits

  • Maximum Memory: 64-bit address space (theoretical)
  • Minimum RAM: 512 MB
  • Recommended RAM: 1 GB or more
  • CPU Architecture: x86_64 only
  • Boot Method: Legacy BIOS (UEFI planned)

Build System

Dependencies

  • Compiler: GCC (x86_64-elf-gcc recommended)
  • Assembler: NASM (Netwide Assembler)
  • Linker: GNU LD
  • Build Tool: GNU Make
  • Emulator: QEMU (for testing)
  • ISO Creation: GRUB2 tools, xorriso

Build Targets

make # Build kernel binary make iso # Create bootable ISO make run # Run in QEMU make debug # Run with GDB debugging make clean # Clean build files make install # Install to system (planned)

Educational Value

Kronos OS serves as an excellent educational resource for understanding:

  • Operating System Concepts: Kernel design, memory management, process scheduling
  • Low-level Programming: Assembly language, hardware interaction
  • System Architecture: x86_64 architecture, boot process, interrupts
  • GUI Development: Graphics programming, window management
  • Driver Development: Hardware abstraction, device drivers
